# Plan: Add Delivery Partners Block in Page Builder

## Goal
Implement your screenshot design as a **new Builder block** inside `app/Filament/Resources/Pages/PageResource.php`, so the section is created and edited from Custom Pages.

## Current Codebase Findings
- `PageResource` already uses `Builder::make('content')` with multiple blocks (`hero`, `rich_text`, `image_text`, etc.).
- Each block supports optional button fields in a consistent pattern.
- Frontend custom-page rendering is through `routes/web.php` catch-all route and `resources/views/pages/show.blade.php`.
- Delivery partner data model exists in `App\Models\DeliveryPartner` with `name`, `image`, `phone`, `is_active`.

## Updated Implementation Direction
1. Add a new Builder block in `PageResource` named `image_cards_section` (or `online_ordering_partners`).
2. Make it **manual image-card only** (as requested): cards are uploaded directly in the block.
3. Render the block in `resources/views/pages/show.blade.php` with screenshot-style layout.
4. Do not depend on `DeliveryPartner` model for this section.

## Block Schema (for `PageResource`)
Proposed fields inside new `Block::make('image_cards_section')`:
- `title` (TextInput, required)
- `subtitle` (TextInput)
- `description` (Textarea)
- `cards` (Repeater, required, min 1):
  - `image` (FileUpload, image, `public` disk, e.g. `pages/cards` directory)
  - `alt` (TextInput, optional)
  - `link` (TextInput url, optional)

This keeps the section exactly as an uploadable image-card block inside Page Builder.

## Frontend Rendering Plan
In `resources/views/pages/show.blade.php` block switch/loop:
- Add case for `image_cards_section`.
- Use title/subtitle/description text from block data.
- Loop through `cards` repeater data.
- Render screenshot-style cards:
  - 1 column mobile, 2 columns tablet+, with spacing similar to screenshot
  - rounded card, soft border, gradient overlay (light top -> dark bottom)
  - centered image (`object-contain`) and bottom arrow icon
  - card wrapped in `<a>` only when `link` exists
  - graceful fallback if image missing (skip card or show placeholder block)

## Validation/UX Rules
- Ensure `title` required so section always has heading.
- Ensure `cards` repeater has at least one entry.
- Add placeholder Bangla defaults matching screenshot tone.
- Guard against empty cards: hide grid and show subtle empty hint in preview/front page.

## Verification Steps (Implementation Phase)
1. Create/Edit a custom page from Filament Pages resource and add the new block.
2. Upload card images in repeater and save page.
3. Verify optional card links open correctly.
4. Check responsive layout on mobile and desktop.
5. Run `vendor/bin/pint --dirty --format agent` (if PHP changed).
